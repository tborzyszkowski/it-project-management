# Zadania – Część 2B: Daily Scrum

> **Czas:** 10 minut | **Prowadzi:** Scrum Master  
> Odbywa się po **22 minutach** sprintu. Wszyscy wstają od komputerów.

---

## Zadanie SM-03 – Przeprowadzenie Daily Scrum

**Wykonuje: Scrum Master**

### Krok 1 – Zatrzymaj sprint (0:00)

Głośno zakomunikuj: **„Czas na Daily Scrum! Wszyscy wstają od komputerów!"**

Odejdź od biurek – stańcie w kółku lub przy tablicy.

---

### Krok 2 – Przejdź przez 3 pytania dla każdego Developera (0:01–0:07)

Zadaj każdemu Developerowi kolejno 3 pytania. Odpowiedź powinna być **krótka (max 30 sekund)**:

#### Pytania

| # | Pytanie | Przykładowa odpowiedź |
|---|---|---|
| 1 | **Co zrobiłeś/-aś od ostatniego spotkania?** | „Napisałem profil detektywa, 180 słów. Czeka na review." |
| 2 | **Co zamierzasz zrobić do następnego?** | „Zacznę pisać Rozdział 1 – scenę otwierającą." |
| 3 | **Czy coś Ci przeszkadza?** | „Nie wiem jak opisać klimat noir – potrzebuję przykładu." |

> ⚠️ Jeśli odpowiedź jest zbyt długa – przerwij i zaproponuj rozmowę **po** Daily Scrum.

---

### Krok 3 – Zbierz impedimenty (0:07–0:09)

Dla każdego zgłoszonego impedimentu:

1. Wróć do komputera i utwórz Work Item w ADO:
   - **Boards → New Work Item → Impediment**
   - Tytuł: np. `"DEV1 nie ma przykładów stylu noir"`
   - Opis: co blokuje, kto zgłosił, kiedy
   - Assigned To: Scrum Master
   - Status: Active

2. Spróbuj natychmiast usunąć przeszkodę:
   - Problem z AI → podaj link do Copilot
   - Problem z ADO → sprawdź uprawnienia
   - Problem merytoryczny → zaproponuj pair-writing z innym Developerem

---

### Krok 4 – Zakończ i wznów sprint (0:09–0:10)

Zakomunikuj: **„Daily Scrum zakończony. Wracamy do pracy! Mamy jeszcze ~23 minuty sprintu."**

---

## Arkusz Daily Scrum (do wypełnienia podczas spotkania)

| Osoba | Rola | Co zrobiłem? | Co zrobię? | Impediment? |
|---|---|---|---|---|
| | Developer | | | |
| | Developer | | | |
| | Developer | | | |
| | PO | *(opcjonalnie)* | *(opcjonalnie)* | |

> SM może wydrukować ten arkusz lub wypełniać w notatniku.

---

## Czego unikać podczas Daily Scrum

| ❌ Błąd | ✅ Poprawne |
|---|---|
| Raportowanie do managera / prowadzącego | Synchronizacja między Developerami |
| „Mam problem z X, może Y by pomogło, albo Z..." | „Nie mogę zrobić X – impediment." |
| Dyskusja techniczna przy całym zespole | Umów rozmowę po Daily Scrum |
| Siadanie przy komputerze podczas mówienia | Stoisz, patrzysz na zespół |
| Przekraczanie 15 minut | SM przerywa i przenosi dyskusję |

---

## Refleksja po Daily Scrum

Prowadzący po zakończeniu Daily Scrum może zapytać:

1. „Jak czuliście się z tym formatem? Co było trudne?"
2. „Czy 3 pytania to wystarczające narzędzie synchronizacji?"
3. „Wyobraźcie sobie – codziennie, o tej samej godzinie. Jakie zalety?"
